# Proposal: A Continuously Updated, Search-Driven UMLS

This proposal frames UMLS 2.0 as a replacement for the static UMLS distribution model, not a rejection of the standard vocabularies that make UMLS useful. The system would continue to ingest authoritative vocabularies such as SNOMED CT, RxNorm, LOINC, MeSH, and other available source terminologies, preserving their identifiers, hierarchies, semantic types, definitions, mappings, and provenance. UMLS would remain the identity and crosswalk layer: every concept, synonym, source code, local extension, and evidence record should resolve to a stable concept identity or to a clearly marked provisional `NEW#######` concept when no adequate standard concept exists.

On top of that foundation, the system would maintain a persistent search and evidence layer that continuously scans biomedical literature, clinical text when permitted, open-access reference sources, and structured biomedical datasets for emerging terms, synonym usage, relationship evidence, and vocabulary gaps. These sources would be represented as evidence documents, vectors, lexical indexes, and relation tables so the system can find concepts by meaning rather than only by exact source-vocabulary strings. Candidate updates would include new synonyms, improved preferred labels, mappings to existing source codes, broader/narrower/related relationships, semantic type assignments, and new local concepts for clinically useful ideas missing from the baseline vocabulary set; each candidate would be evaluated using evidence strength, semantic consistency, provenance, and measured impact on retrieval quality.

Updates would be handled as an iterative, versioned decision process. High-confidence additions could be applied automatically, uncertain changes could be routed to expert review, and every change would remain reversible through explicit delta files and build manifests. Indexing would be incremental rather than rebuilt from scratch, using layered stores that separate stable source-vocabulary content from rapidly changing evidence, labels, relationships, and local concepts. The result would be a living, auditable, openly searchable terminology platform: grounded in authoritative vocabularies, extensible through evidence-backed `NEW#######` concepts, responsive to new biomedical language, and capable of feeding validated terms and relationship signals back to source vocabularies such as MeSH.
