How To Read The Buckets

Approved / Promoted

Evidence that passed a measured gate and improved or preserved the target without unacceptable regression. This evidence can be used for the approved path named in the row.

Tested Neutral / Candidate

Evidence that was evaluated and did not break the focused gate, but either did not improve enough or needs routing/filtering before broad promotion.

Tested Rejected

Evidence or a build/linking path that failed a gate. The artifact may stay on disk for audit, but it should not be reused as default evidence without a new hypothesis.

Configured / Not Yet Live-Tested

Benchmark rows or source-specific lanes that exist but still need a live API run and pass/fail record before they can become routine gates.

Inventory / Diagnostic Only

Built bundles, external benchmarks, logs, or enrichment artifacts that are useful for review but not product-approved evidence by themselves.

Source Inventory Buckets

Source: source evidence dashboard. These buckets describe current artifact/source role, not approval by themselves.

Bucket Sources Meaning Use Safely As
Base/reference evidence UMLS, HPO, MONDO, PubMed systematic pilot, MedlinePlus systematic snapshot Core reference or systematic/pilot sources already represented in the product inventory. Trusted source families, still subject to benchmark gates and source-specific tests.
Partial/subset evidence PMC OA, Europe PMC, DailyMed, MedlinePlus Genetics, NCBI Bookshelf OA, NCI, FDA, CDC, NIDDK Authoritative or useful sources with subset, topic-harvest, sample, or outcomes-only coverage. Scoped evidence; do not claim systematic coverage unless a source-specific gate says so.
Association signal / opt-in CUI2Vec, BioConceptVec, ClinicalTrials.gov posted outcomes, PubTator3 sampled relation candidates Useful candidate signals that can introduce drift, require narrower evidence semantics, or need literature validation. Use only in measured probes or explicit exploration modes; PubTator3 remains a stub until validated against PubMed/PMC for a measured gap.
Enrichment / candidate only OpenAlex Enrichment source with tested crowding risk in default search. Candidate or routed enrichment, not broad default evidence.
Diagnostic only Local search logs, reviewed snippets Useful for evaluation, triage, and examples, but not authoritative source evidence. Test material or audit material, not source authority.
Needs provenance decision Local extension concepts / mixed public bundle Generated or mixed artifacts that need source/provenance boundaries before source claims. Only use with explicit provenance and public-output policy checks.

Maintenance Rule

When a future iteration acquires, tests, promotes, rejects, or changes routing for evidence, update this page in the same iteration. A file existing under build/ is not enough to move it out of candidate, diagnostic, or untested status.
