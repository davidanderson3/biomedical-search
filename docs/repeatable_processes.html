Last updated: 2026-06-10

Scope: this covers processes that are currently documented or backed by scripts. It does not replace the specialized docs; it tells you which repeatable process to use, which command starts it, what inputs it expects, what artifacts it writes, and how to verify the result.

source-code failure.

  2. Record the baseline result before changing anything.
  3. Choose exactly one route:

  • code route: change parsing, ranking, filtering, source-code output, UI, or
  • evidence route: procure or integrate authoritative, license-compatible

regression in false positives, assertion status, source policy, code output, or held-out/standing smoke.

  1. Rebuild/restart only what the route requires.
  2. Re-run the same benchmark slice plus the required smoke tier.
  3. Promote only if the target weakness improves and there is no unacceptable
  4. If the route does not improve things, reject it:

  • code route: revert or remove the change before it enters the shipped path;
  • evidence route: exclude the evidence from the default/search-ranking path

Decision: Keep, Decision: Revise, or Decision: Revert.

with decision_rule set to promote, no_regression, or diagnostic, and decision set to accept, neutral, or reject.

  • For evidence changes, add or update config/source_acquisition_progression.tsv
