Evidence Acquisition Gate

Use this page only after a concrete search weakness has been reproduced. The first decision is whether more source evidence is actually the fix.

Stop first Not an evidence path

If the right source appears but is ranked low, filtered out, poorly hydrated, mistranslated, or mislabeled in evaluation, fix search behavior before acquiring evidence.

Evidence path Acquire or rebuild

Use source acquisition when the weakness lacks a credible source row, has only reference/enrichment support, or depends on a source slice that is known to be partial.

Authority rule Start narrow

Prefer the most authoritative source for the weakness, preserve provenance, and add a focused query or judgment so the new evidence can prove its value.

Current coverage 2 systematic / 11 partial

Systematic products: PubMed, MedlinePlus. Treat topic and subset products as candidates, not coverage claims.

Coding Change vs Evidence Acquisition

When I work a search weakness, I will name the path before changing code or acquiring sources.

Choose coding change

The right evidence exists but search mishandles it

  • Correct source rows appear below the fold or behind worse rows.
  • Hydration, filters, public-output rules, language handling, or display labels suppress the right result.
  • The weakness is reproducible with existing artifacts and can be fixed by ranking, parsing, indexing, or API/UI behavior.

Default action: Patch code or configuration, then rerun the focused query and regression/smoke checks.

Choose evidence acquisition

The source material is missing, too thin, or not trustworthy enough

  • No credible source row exists for the expected concept, relation, passage, or source family.
  • Only ontology, enrichment, local notes, or protocol text supports a claim that needs primary/public evidence.
  • The current source slice is explicitly topic/subset/demo scale and the weakness requires broader or different coverage.

Default action: Acquire or rebuild the narrowest authoritative source slice, preserve provenance, and add a focused judgment.

Choose dual path

Both evidence and code are blocking the weakness

  • The source is partially present but incomplete, and ranking/filtering also fails on the present rows.
  • A new source slice will not prove useful unless search behavior changes at the same time.
  • Evaluation cannot distinguish source absence from retrieval failure without a focused test case.

Default action: State both decisions, sequence the lower-risk fix first, and keep acquisition scoped to the reproduced weakness.

Report format

Decision: coding change, evidence acquisition, or dual path. Reason: the observed weakness and the evidence/code signal. Verification: the query, artifact, or test that proves the choice.

Weakness to Source Router

Pick the source family from the observed weakness, not from whichever artifact is easiest to fetch.

Term, synonym, or CUI is missing

Start withUMLS, HPO, MONDO
Then checkPublic-safe label policy and display-name checks

Use when: The concept fails lookup or normalization before ranking has a chance.

Avoid when: The concept is retrieved but loses after ranking, filters, or UI hydration.

Biomedical concept lacks article-level support

Start withPubMed
Then checkPMC Open Access or Europe PMC for full text

Use when: The weakness needs PMID provenance, disease/drug/gene/procedure literature, or guideline/case-report support.

Avoid when: The issue is consumer wording, regulatory wording, or a known long-document ranking failure.

Abstracts are too thin or long-document mentions are missed

Start withPMC Open Access
Then checkNCBI Bookshelf OA for review-style passages

Use when: The needed evidence is likely in body text, mechanisms, paragraphs, tables, or review chapters.

Avoid when: A short abstract already contains the evidence and the ranker is failing to surface it.

Drug label, warning, contraindication, or dosage context is weak

Start withDailyMed
Then checkFDA, then PubMed for supporting literature

Use when: The expected answer depends on SPL label sections or public regulatory context.

Avoid when: The claim is comparative efficacy or off-label outcome evidence.

Patient-language query does not match clinical terminology

Start withMedlinePlus
Then checkMedlinePlus Genetics for genetic language

Use when: The result should bridge consumer phrasing to a known condition, drug, procedure, or symptom.

Avoid when: The weakness is a missing CUI, ontology relation, or article-level biomedical proof.

Genetics, inheritance, gene-condition, or phenotype context is weak

Start withMedlinePlus Genetics
Then checkHPO, MONDO, PubMed

Use when: The query depends on gene names, inherited conditions, phenotypes, or patient-readable genetics context.

Avoid when: The weakness is general disease coverage without a genetics angle.

Cancer disease, staging, or oncology treatment context is weak

Start withNCI
Then checkPubMed or PMC OA for literature support

Use when: The expected source should be cancer-specific and public/authoritative.

Avoid when: The topic is not oncology-specific or only needs terminology normalization.

Public health, prevention, epidemiology, or population guidance is weak

Start withCDC
Then checkNIDDK, FDA, or PubMed depending on domain

Use when: The weakness is guidance, prevention, surveillance, outbreak, or population-facing context.

Avoid when: The expected answer is individual treatment efficacy or terminology lookup.

Treatment effect or trial-result evidence is weak

Start withClinicalTrials.gov outcomes only
Then checkPubMed randomized trials or guidelines

Use when: Posted outcome results can answer the weakness and protocol-only text is excluded.

Avoid when: Eligibility, recruitment, planned endpoints, or registry text would be the only available support.

Relationship edge or co-mention is weak

Start withPubTator3 as candidate discovery
Then checkPubMed or PMC OA validation

Use when: The weakness is an entity/relation candidate that should be tested against literature evidence.

Avoid when: Automated annotation alone would become the final source of truth.

Need to prioritize what literature to acquire next

Start withOpenAlex
Then checkPubMed or PMC OA acquisition

Use when: Citation metadata can identify high-value gaps or missing papers.

Avoid when: Using citation metadata as primary biomedical evidence.

Real query pressure or reviewer observation is unclear

Start withLocal Search Logs and Reviewed Snippets
Then checkConvert to benchmark rows before source acquisition

Use when: The weakness needs a reproducible query, expected concept, and expected source family.

Avoid when: Treating logs or notes as source evidence.
